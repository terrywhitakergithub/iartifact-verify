Who is this for? If someone sent you an iArtifact — a piece of music, visual art, writing, software, photography, film, a beat, a physical work, an NFT, a patent filing, a trademark, or any creative or professional record — this tool lets you independently confirm it is exactly what they say it is. You don't need an account. Nothing leaves your machine. Verification runs entirely in your browser using the same SHA-256 hashing the iArtifact app used when the record was created.

What does it prove? That every file is byte-for-byte identical to what was recorded at creation, that the provenance chain linking every event is mathematically intact, and that the RFC 3161 timestamp was signed by a trusted third party — not the creator.

How verification works
01
File Integrity
Every file is re-hashed using the browser's native Web Crypto API (SHA-256) and compared against the value stored in manifest.json at creation time. Any alteration — even one byte — fails this check.
02
Provenance Chain
Every event (create, scan, mint, stamp, seal) is stored in chain.json as a linked hash entry. This verifier recomputes each entry_hash and confirms it correctly links to the previous one. Tamper any entry and the chain breaks.
03
iA-ID Confirmation
The iA-ID is derived from the creation entry hash in the chain. This verifier confirms the iA-ID matches — proving the artifact's identity has not been substituted or cloned from another record.
04
RFC 3161 Timestamp
The .tsr token issued by freetsa.org is a signed timestamp that cannot be backdated. It proves the file existed at that exact moment — signed by a third party's private key that no one else holds.
